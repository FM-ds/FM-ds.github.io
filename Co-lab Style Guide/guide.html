UK Growth Diagnostics

Graphics Guidelines

This document outlines rules and best practices for charts, maps, and diagrams made for the UK Growth Diagnostics report.

Accompanying Vega-lite and Altair themes are provided.

Axes

  1. Axis titles. In most cases, axis titles can be avoided. For example, a chart of investment by UK region over time would not need a y-axis title. Instead, the topic and measure should be defined in the chart title and subtitle, and axis labels should be appropriately set. Date and time axes should never need a title. In some cases (e.g. scatter plots), it may be necessary to add titles.
  2. Labels Axis lables should include units where appropriate. Time axes should only include required detail - a chart spanning a decade does not need month labels.
    How to Achieve:

    Make sure labels are defined descriptively. You can use label expressions to set custom label formatting:

    
"axis": {"labelExpr": "'Â£'+datum.value + 'B'"}

Panel charts are useful for comparing a single indicator across multiple countries. Care should be taken over the number of series represented. For example a gender breakdown for 8 countries would mean 16 series, which is likely too many to be useful. In this case, consider a crossectional bar chart or a small multiples chart.

  1. Countries. When making international comparisons, use the consistent set of peer countries: United Kingdom, France, Italy, Spain, Germany, Japan, Canada, and the United States.
  2. Labels. Prefer end labels over legends.
    How to Achieve:

    End labels can be implemented by adding a text-layer filtered to the final year of data for each country.

    
"layer" : [
...
{
    "transform": [
        {"filter": {"field": "year", "equal": 2022}} // filter to the a single year of data
    ]
    "mark": {"type": "text", "align": "left", "baseline": "middle", "dx": 3},
    "encoding": {
        "text": {"field": "value", "type": "quantitative"},
        "x": {"value": 450}, // Setting a fixed x position means that the labels will be aligned, even if the series end at different points
        "y": {"field": "value", "type": "quantitative"},
        "color": {"field": "country", "type": "nominal"}
    },
}
]
  3. Highlighting the UK. The UK is the primary focus of the report, and should be highlighted in all charts. It lines should be thicker, not dashed, and appear above other lines.
    How to Achieve:

    Use conditions for the stroke-dash, and size. Sort the UK last in the underlying data to make its lines appear on top. 

    
"layer" : [
{
...
"encoding" : {
    "size": {"condition": {"test": "datum.country == 'United Kingdom'", "value": 2}, "value": 1.5},
    "strokeDash": {"condition": {"test": "datum.country == 'United Kingdom'", "value": [0, 0]}, "value": [5, 2]},
}
...
}
]
